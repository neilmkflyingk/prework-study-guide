HTML

  • HTML creates the framework for the web application.
  • The head element contains information, or metadata, that helps the browser to render the page.
  • Metadata is data about our webpage that the browser uses to effectively serve our webpage to users. it is typically not seen by users, but it contains iportant information about accessibility, search engines, and performance
  • The body element contains the content that is shown to the user. All the elements that we want the user tp see and interact with must be placed inside this element.
  • An HTML attribute defines additional information about an element. They are always defined in the beginning element.
  • The title element defines the title for a webpage. This element is required and is extremely important for Search Engine Optimization (SEO).
  • SEO is the process of improving the ranking of a website in search engines such as Google or Bing. Better SEO results in your website more frequently showing up in searches.
  • The style element defines the CSS styles associated with a website.
  • The base element defines the base URL for a webpage.
  • The link element connects an external resource to the HTML document.
  • The meta element defines metadata such as the character set, description, keywords, author, and viewport.
  • The script element embeds client-side scripts in the HTML document.
  • In semantic elements, the name of the element describes its content and the role it plays on a webpage. Body,header, main and footer are examples of semantic elements.
  • Child elements are those elements located within other elements (parent elements).
  • The footer element appears at the bootom of the page and usually contains the author, copyright, sitemap, and navigation.

CSS

  • CSS pertains to colors, aesthetics, visual layout, land organization
  • Inline CSS is used to style an element directly in the HTML file. They apply to specific HTML tags.
  • Internal style sheet goes in the head element. It is used to style multiple elements in the page code.
  • External CSS style sheet keeps all CSS rules in a separate file. The file is then linked in HTML head element
  • A margin indicates how much space we eant around the outside of an element.
  • A padding indicates how much space we eant around the content inside an element.
  • CSS Rule Breakdown: The selector defines the element or attributes to which the rules, or declarations, will apply. Declarations contain the property we want to apply and the value of the property.
  • Class Attributes allow us to share a CCSS rule to any element we choose by assigning the rule tp a class attribute with a class selector. Then the class attribute can be assigned tp the element in the HTML, enabling us to freely choose which elements will be assigned the CSS rule. The . Preceding the selector designates it as a class selector.

Git

Terms

  • Git- Version control system that lets you manage and keep track of each version of your project. Accessed through the command line.
  • GitHub- Cloud based hosting service used for version control, debugging, and collaboration between developers remotely.
  • Repository (repo) - a folder in which all files and their version histories are stored.
  • Branch - a workspace in which you can make changes that won't affect the live site.
  • Markdown (.md) - a way to write in github that converts plain text to github code. Sites such as Atom and Sublime Text are examples of free resources for developers using markdown
  • Commit Changes - a saved record of a change made to a file within the repo.
  • Pull Request (PR) - the way to ask for changes made to a branch to be merged into another branch that also allows for multiple users to see, discuss and review work being done.
  • Merge - after a pull request is approved, the commit will be pulled in (or merged)from one branch to another and then, deployed on the live site.
  • Issues - How work is tracked when using git. Issues allow users to report new tasks and content fixes, as well as allow users to track progress on a project board from beginning to end of a specific project.
  • Federalist - a platform that securely deploys a website from github repository in minutes and lets users preview proposed and published changes.

Git Flow

  1. Navigate to the correct working directory.
  2. Make sure you are in the current main branch. (git checkout main)
  3. Pull down the latest main branch. (git pull origin main)
  4. Create feature branch to work within (git checkout -b feature/add-name)
  5. Confirm branch. (git status)
  6. Complete work on the project.
  7. Stage your changes. (git add-A)
  8. Commit changes to repo. (git commit -m “brief note about commit”)
  9. Confirm your current branch is up to date. (git pull origin main)
  10. Push up changes. (git push origin feature/add-name)
  11. Open repo in GitHub.
  12. Navigate to pull requests, click new pull request.
  13. Confirm you are comparing main branch and your current feature branch
  14. Give PR a title and a short description of changes.
  15. Click “Create pull request” button.
  16. Click “Merge pull request” button.
  17. Navigate to main branch to confirm changes.
  18. Update local repo. (git checkout main) (git pull origin main)
  19. Close issue for work if applicable.
